Market Basket Analysis — Apriori

Find which items are frequently bought together
Association RulesUnsupervisedRetail / E-commerce
Aim
To discover frequent itemsets and generate association rules from transactional data to understand customer purchasing patterns and improve product recommendations.
Problem understanding
Given a dataset of customer transactions (e.g., supermarket receipts), we want to find items that co-occur frequently. Example: "80% of customers who buy bread also buy butter." Three key metrics — support (how often an itemset appears), confidence (how often the rule is correct), and lift (how much better than random chance) — drive the analysis.
Algorithm steps
  1. Load and encode transaction data into a binary matrix (1 = item bought, 0 = not bought)
  2. Set minimum support threshold (e.g., 0.05 = item appears in 5% of transactions)
  3. Run Apriori to find all frequent itemsets meeting the support threshold
  4. Generate association rules from frequent itemsets
  5. Filter rules by minimum confidence (e.g., 0.5) and lift (lift > 1 means positive association)
  6. Interpret top rules: {antecedent} → {consequent}
R code
library(arules)
library(arulesViz)

# Load dataset (use built-in or CSV)
data("Groceries")  # built-in dataset

# Or load your own:
# transactions <- read.transactions("basket.csv", format="basket", sep=",")

# Run Apriori
rules <- apriori(Groceries,
                 parameter = list(supp = 0.05,
                                  conf = 0.5,
                                  minlen = 2))

# View top rules sorted by lift
rules_sorted <- sort(rules, by = "lift")
inspect(head(rules_sorted, 10))

# Visualize
plot(rules_sorted, method = "graph", 
     control = list(type = "items"))

# Summary
summary(rules)
How to interpret results
Support = 0.08 → This itemset appears in 8% of all transactions
Confidence = 0.75 → When item A is bought, 75% of the time item B is also bought
Lift = 2.3 → This combination is 2.3x more likely than by random chance (lift > 1 is good)

Tell the examiner: "We set support = 0.05 to filter rare items and confidence = 0.5 to keep only strong rules. Lift > 1 confirms a genuine positive association."

Customer Segmentation — k-Means Clustering

Group customers based on similar behavior
ClusteringUnsupervisedMarketing / CRM
Aim
To segment customers into distinct groups based on purchasing behavior (e.g., RFM: Recency, Frequency, Monetary) so that targeted marketing strategies can be applied to each group.
Problem understanding
k-Means partitions n data points into k clusters. Each point belongs to the cluster with the nearest mean (centroid). We don't know the labels beforehand — it's unsupervised. The key challenge is choosing the right k, done using the Elbow Method (plot within-cluster sum of squares vs k and find the "elbow" bend).
Algorithm steps
  1. Load and preprocess data — normalize/scale features using scale()
  2. Use Elbow Method to find optimal number of clusters k
  3. Run k-Means with chosen k
  4. Assign cluster labels back to original data
  5. Visualize clusters using ggplot2 or factoextra
  6. Profile each cluster — compute mean values per cluster to describe them
R code
library(ggplot2)
library(factoextra)

# Load data
data <- read.csv("customers.csv")  
# Or use: data(iris) for practice

# Select numeric features and scale
df <- scale(data[, c("Age", "Annual_Income", "Spending_Score")])

# Elbow method to find optimal k
fviz_nbclust(df, kmeans, method = "wss") +
  labs(title = "Elbow Method")

# Run k-Means with k=3 (or your chosen value)
set.seed(42)
km <- kmeans(df, centers = 3, nstart = 25)

# Add cluster to original data
data$Cluster <- as.factor(km$cluster)

# Visualize
fviz_cluster(km, data = df,
             palette = c("#2E9FDF", "#00AFBB", "#E7B800"),
             geom = "point",
             ellipse.type = "convex",
             ggtheme = theme_bw())

# Profile clusters
aggregate(data[, c("Age","Annual_Income","Spending_Score")],
          by = list(Cluster = data$Cluster), FUN = mean)
How to interpret results
Cluster 1 → High income, high spending = "Premium customers" — target with loyalty programs
Cluster 2 → Low income, low spending = "At-risk customers" — target with discounts
Cluster 3 → High income, low spending = "Potential customers" — target with awareness campaigns

Tell the examiner: "We scaled the data to prevent features with large ranges from dominating. We chose k=3 based on the elbow in the WSS plot. Each cluster represents a distinct customer segment with actionable marketing implications."

Disease/Survival Prediction — SVM + Cox Regression

Classify disease presence and predict survival time
ClassificationSupervisedHealthcare
Aim
SVM: To classify patients as disease-positive or negative based on clinical features.
Cox Regression: To model the relationship between patient features and time-to-event (survival time), predicting risk of death or relapse over time.
Problem understanding
SVM finds a hyperplane that best separates two classes with maximum margin. Kernel trick (e.g., RBF kernel) handles non-linear data. Cox Regression is used for survival analysis — it models the hazard rate (risk of event at time t) as a function of covariates. Useful when some patients are censored (event not yet observed).
Algorithm steps — SVM
  1. Load clinical dataset, split into train (70%) and test (30%)
  2. Normalize features using scale()
  3. Train SVM with RBF kernel using e1071 package
  4. Predict on test set, generate confusion matrix
  5. Report accuracy, sensitivity, specificity
R code — SVM
library(e1071)
library(caret)

# Load data (example: heart disease dataset)
data <- read.csv("heart.csv")
data$target <- as.factor(data$target)

# Train-test split
set.seed(42)
idx <- createDataPartition(data$target, p=0.7, list=FALSE)
train <- data[idx, ]
test <- data[-idx, ]

# Scale features
train_s <- train
test_s <- test
train_s[,-ncol(train)] <- scale(train[,-ncol(train)])
test_s[,-ncol(test)] <- scale(test[,-ncol(test)])

# Train SVM
svm_model <- svm(target ~ ., data = train_s,
                 kernel = "radial", cost = 1, gamma = 0.1)

# Predict
pred <- predict(svm_model, test_s)
confusionMatrix(pred, test_s$target)
R code — Cox Regression
library(survival)
library(survminer)

# Load survival data (time = survival time, status = 1 if event occurred)
data(lung)  # built-in survival dataset

# Fit Cox model
cox_model <- coxph(Surv(time, status) ~ age + sex + ph.ecog + wt.loss,
                   data = lung)

# Summary of model
summary(cox_model)

# Visualize survival curves
fit <- survfit(cox_model, newdata = data.frame(
  age=60, sex=1, ph.ecog=1, wt.loss=0))
ggsurvplot(fit, data = lung,
           conf.int = TRUE,
           ggtheme = theme_minimal(),
           title = "Cox Regression Survival Curve")
How to interpret results
SVM: Accuracy = 87% means the model correctly classifies 87% of patients. AUC close to 1 = excellent discriminating power.

Cox: HR (Hazard Ratio) > 1 means the variable increases risk of the event. E.g., HR = 1.8 for smoking means smokers have 80% higher risk. p-value < 0.05 means the variable is statistically significant.

Tell the examiner: "SVM maximizes the margin between classes using a hyperplane. The RBF kernel handles non-linearity. Cox regression extends survival analysis by allowing multiple covariates and handles censored data."

Sales/Price Prediction — Linear Regression

Predict a continuous numeric outcome
RegressionSupervisedSales / Finance
Aim
To build a predictive model that estimates a continuous target variable (e.g., house price, sales revenue) based on one or more input features, and evaluate model accuracy using RMSE and R-squared.
Problem understanding
Linear Regression models the linear relationship: Y = b0 + b1*X1 + b2*X2 + ... + e. The model learns coefficients (b) that minimize the sum of squared residuals (errors). R² tells us how much variance in Y is explained by the model. RMSE tells us the average prediction error in the same unit as Y.
Algorithm steps
  1. Load dataset, explore with summary() and correlation matrix
  2. Handle missing values, outliers, encode categorical variables
  3. Split into train (70%) and test (30%)
  4. Fit linear regression model using lm()
  5. Interpret coefficients and p-values
  6. Predict on test set, calculate RMSE and R²
  7. Plot actual vs predicted values to visualize performance
R code
library(ggplot2)
library(caret)

# Load data (example: house prices)
data <- read.csv("housing.csv")
# Or use built-in: data(mtcars)

# Explore
summary(data)
cor(data[, sapply(data, is.numeric)])

# Train-test split
set.seed(42)
idx <- createDataPartition(data$Price, p = 0.7, list = FALSE)
train <- data[idx, ]
test <- data[-idx, ]

# Fit linear model
model <- lm(Price ~ Area + Bedrooms + Age + Location, data = train)

# Model summary
summary(model)

# Predict
pred <- predict(model, test)

# Evaluate
rmse <- sqrt(mean((test$Price - pred)^2))
r2 <- cor(test$Price, pred)^2
cat("RMSE:", round(rmse, 2), "\n")
cat("R-squared:", round(r2, 4), "\n")

# Plot actual vs predicted
plot_df <- data.frame(Actual = test$Price, Predicted = pred)
ggplot(plot_df, aes(x = Actual, y = Predicted)) +
  geom_point(color = "#2E9FDF", alpha = 0.6) +
  geom_abline(slope=1, intercept=0, color="red", linetype="dashed") +
  labs(title = "Actual vs Predicted", x = "Actual Price", y = "Predicted Price") +
  theme_minimal()
How to interpret results
R² = 0.85 → The model explains 85% of the variance in price. Closer to 1 is better.
RMSE = 15000 → Average prediction error is ₹15,000
Coefficient of Area = 520 → For every 1 sq ft increase, price increases by ₹520
p-value < 0.05 → That predictor is statistically significant

Tell the examiner: "We checked assumptions: linearity, normality of residuals, and no multicollinearity (VIF < 5). The model is validated on unseen test data to avoid overfitting."

Churn/Spam Classification — Logistic Regression + Decision Tree

Classify binary outcomes (yes/no, churn/stay)
ClassificationSupervisedTelecom / Email
Aim
To predict whether a customer will churn (leave) or an email is spam based on input features, using Logistic Regression for probability-based classification and Decision Tree for rule-based interpretable classification.
Problem understanding
Logistic Regression: Models the probability of a binary outcome using the sigmoid function. Output is between 0 and 1; threshold (usually 0.5) converts probability to class.
Decision Tree: Recursively splits data using the best feature at each node (based on Gini impurity or Information Gain) until leaves are pure or max depth is reached. Very interpretable — shows "if-then" rules.
Algorithm steps
  1. Load data, handle imbalanced classes if needed (use SMOTE or class weights)
  2. Encode categorical variables, split into train/test
  3. Train Logistic Regression with glm(family="binomial")
  4. Train Decision Tree with rpart()
  5. Evaluate both using confusion matrix, accuracy, AUC-ROC
  6. Visualize Decision Tree — show rules clearly
  7. Compare both models: when to use which
R code — Logistic Regression
library(caret)
library(pROC)

# Load data
data <- read.csv("churn.csv")
data$Churn <- as.factor(data$Churn)

# Train-test split
set.seed(42)
idx <- createDataPartition(data$Churn, p=0.7, list=FALSE)
train <- data[idx, ]
test <- data[-idx, ]

# Logistic Regression
logit_model <- glm(Churn ~ ., data = train, family = "binomial")
summary(logit_model)

# Predict probabilities
prob <- predict(logit_model, test, type = "response")
pred_class <- ifelse(prob > 0.5, "Yes", "No")
pred_class <- as.factor(pred_class)

# Evaluate
confusionMatrix(pred_class, test$Churn)

# ROC Curve
roc_obj <- roc(test$Churn, prob)
plot(roc_obj, main = "ROC Curve - Logistic Regression")
cat("AUC:", auc(roc_obj), "\n")
R code — Decision Tree
library(rpart)
library(rpart.plot)

# Train Decision Tree
tree_model <- rpart(Churn ~ ., data = train,
                    method = "class",
                    control = rpart.control(maxdepth = 5,
                                            minsplit = 20))

# Visualize tree
rpart.plot(tree_model, type = 4, extra = 101,
           main = "Decision Tree — Churn Prediction")

# Predict
tree_pred <- predict(tree_model, test, type = "class")

# Evaluate
confusionMatrix(tree_pred, test$Churn)

# Feature importance
tree_model$variable.importance
How to interpret results
Accuracy = 84% → Model correctly classifies 84% of customers
AUC = 0.89 → Excellent discrimination between churn and non-churn (AUC > 0.8 is good)
Decision Tree rule example: "If contract = Month-to-Month AND tenure < 12 months → Predict CHURN (82% confidence)"
Logistic coefficient = 1.4 for MonthlyCharges → Odds of churn increase by e^1.4 = 4x for each unit increase

Tell the examiner: "Logistic Regression is better when you need probability scores and the relationship is roughly linear. Decision Tree is better when interpretability/rules matter and when there are non-linear interactions. For production, ensemble methods like Random Forest improve on both."
